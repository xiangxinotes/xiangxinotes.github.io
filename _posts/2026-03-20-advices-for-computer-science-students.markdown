作为一个走过弯路、踩过坑的过来人，今天想给计算机专业的后辈5条建议。

不是鸡汤，全是能直接落地的东西。

---

## 学会使用 GitHub，熟练掌握 Git 常用指令

Git 已经从可选技能变成了**计算机专业的基本功**啦。

以我10年工作经验来说，工作中完全离不开 Git。

我 2024 年面试应届毕业生的时候，很多同学连 GitHub 是什么都不知道，让我有点吃惊。

毕竟我毕业的时候就用过 GitHub 了，还和当年的面试官聊过我账号的取名。

如果你也不知道 GitHub 的话，赶快用起来哦。

你不需要学会特别复杂的操作，只要掌握：
- git clone
- git pull
- git add
- git commit
- git push
- git checkout
- 处理简单冲突

就足够在工作中应对一些基础工作了。

GitHub 更是你未来的**线上简历**。
越早开始用，你就越有优势。

## 搭建个人作品集，它比成绩更有说服力
这是我最最最想强调的一条。

**企业招聘不看你成绩单多少分，只看你能做出什么。**

如果你的成绩很好，那就用个人作品集来**锦上添花**；如果你成绩不好，那么个人作品集就是你**逆风翻盘**的机会。

你可以把课程**作业**、自己写的小项目、**毕业设计**作品、**课程设计**作品、学习**笔记**全部放到 GitHub，再用 **GitHub Pages** 展示出来。

这就是你的作品集了。

相信我，项目再小，也比“我会XX语言”这几个字**强100倍**。

我面试应届生时，多希望能在简历里看到作品集链接啊。（可惜我没遇到过这样的学生。

如果有作品集，我一定会去看看，然后和这位学生好好聊一聊。

我对这位学生的**印象也会更深**。

毕竟没有社会经验的学生，如果没有作品集，真的都没什么记忆点。

## 熟悉 Linux 基础命令，非常加分
Linux 是实施、后端、运维、测试、开发最常接触的环境。

不管你有没有学过，我都建议你**掌握 Linux 基础命令**。

你不需要精通内核，只要会：
- 查看文件、目录
- 复制、移动、删除
- 权限操作

就已经超过大部分同学了。

Linux 熟练，**面试更有优势**，实习、工作也会**轻松**很多。

## 想当程序员？成绩好不够，面试能答上来才是王道

我当年，专业课几乎全是 90 / 95 分以上。

但第一次面试时，我被问得哑口无言。

我记得面试官问了一个问题：“什么是数据库**范式**？” 

我根本回答不上来，因为**我不记得了**。

数据库、数据结构与算法是大二学的，所以到了找工作的时候，我已经把概念性的东西忘完了。

我当然有我的理由，我也相信我能十分快速的拾起这些知识，但我面试表现的确实一塌糊涂。

最后也确实没能拿到那份工作机会。

所以，不要只是上课**听讲**，面试前也要**复习**岗位对应的知识，最好也刷**刷题**。

成绩好确实让你的成绩单看起来更好看，但如果你面试的时候回答不上问题，公司根本不会看你的成绩单。

## 关注 AI 发展，用 AI 提升自己

AI 时代已经来了，不会用 AI 的学生会非常吃亏。

你不需要懂算法原理，但你需要：
- 知道 AI 能做什么
- 学习如何写高效的提示词
- 能用 AI 帮你写代码、查bug

把 AI 当成你的超级外挂，
你的效率会直接翻倍。

保持学习，保持好奇，
不要被时代甩开。
